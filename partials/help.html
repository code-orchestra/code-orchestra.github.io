What is COLT

COLT is JavaScript application development software. COLT makes it easier to immediately debug the code. It does so delivering live JS, HTML and assets updates to running application without restarting it. COLT also has run-time code analysis features, helps to write code (code completion and jsdocs), and to inspect JS objects.

What runtime environments does COLT work with

COLT supports popular web-browsers, mobile browsers, node.js and node-webkit.

Getting started with COLT

  1. Download the package for your OS from COLT.IO and install the application (Make sure your computer has Java JRE. You can get it from https://www.java.com/en/download/).
  2. Install the plugin for your code editor.

    3. Sublime Text (2 or 3)

    Install COLT package using Package Control.

    https://sublime.wbond.net/packages/COLT

    Webstorm

    Preferences -> Plugins -> Browse Repositories -> COLT Support

    http://plugins.jetbrains.com/plugin/7435?pr=

  3. In your code editor open or create new project.
  4. Open html document you want to start working with.
  5. Select Tools -> "Run With COLT"

    6. You will need to enter the path to COLT in the following dialog:

    Mac - Applications/COLT/COLT.app

    Win - c:/Programm Files/COLT/colt.exe

    Linx - path to COLT distribution in your system

  6. COLT window will appear briefly and minimize itself.
  7. The browser will load html file.
  8. Make some changes to your html file and save it - your changes will be reflected in the browser.

Possible problems and solutions

  1. The browser tab is open, but the page is "offline" - you can see it grayed out. Try to reload the page. Session icon in COLT's bottom right corner should turn green. Sublime and Webstorm should also indicate that live session has started with ([~]). If it's not helping, try Run with COLT command again.
  2. COLT did never start. Double-check the path to COLT in the editor settings.
  3. COLT did start, but the page did not open. There could be problem with your Java runtime. Check COLT logs tab in COLT UI.

Using COLT with code editors that do not have COLT plugins

COLT does not require you to use particular code editor.

  1. Open COLT app.
  2. Go to File -> New Project
  3. (Important!) Save project file to your project base (root) folder.
  4. Select main html file in the project settings.
  5. Click the "lightning bolt" icon in bottom-right UI corner.
  6. Browser will open the page and "lightning bolt" icon will turn green.
  7. Now edit the project source files in any editor. All the changes should reflect in running application immediately.

Project Settings

Main Document

Path to main document in the project. This is your app entry point.

Can be one of the following.

  1. Path to local HTML document. E.g. /projects/myProject/index.html
  2. Web server URL (live-proxy mode) E.g. http://myserver.com/index.html
  3. Path to JS file (node.js)
  4. Path to main HTML document (node-webkit)

Browser

This is web application. You can specify which browser to open it with. System default browser is used, if you leave it blank.

node.js

Launch node.js server or any application using node.js. You need node.js to be installed in your system. (Note. Mac-OS users will need to specify the path to node.js)

node-webkit

Launch node-webkit application. Please specify the path to node-webkit (node-webkit.app, node-webkit.exe etc) for best experience.

Console

If you are not happy with COLT launchers, you can run your custom command with any parameters. This can be used with phonegap apps.

Manual

You will start the application manually. COLT will wait for client connection when you start the session. For example, you could build phonegap app with xcode or Android Studio and upload to the device. Read more about "launching phonegap apps in live mode on the device".

Use Custom Output

COLT keeps live session files in special folder by default. You can override this with the folder of your choice (warning, do not create "out" folder inside your project folder to avoid infinite recursion)

Exclude from Livecoding Pattern

By default COLT will process all JS/HTML files, except minified scripts. You might want to exclude some files or folders from processing. For example, to improve session start time. You can specify comma-separated list of file and folder names. Paths that start with slash are interpreted as relative to project folder (e.g. /staticLibs)

Max Loop Iterations

To avoid accidental endless loops, COLT injects max iterations count checking code in your loops. Default value is 10000, longer loops will throw exception. Adjust this value if your code needs longer loops.

Don't Catch Runtime Exceptions

You can accidentally add bad code to running app during live session, that can throw runtime exceptions. To prevent this, COLT wraps every function into try/catch. If you rely on exception handling in your code, or you want error stack trace - turn this option off.

Simple Reload Mode

In some cases, e.g. if you work within framework that dynamically transforms DOM you will need to reload the page after every update. Use this option in those cases.

Clear Messages Log on Session Start

By default "Logs" tab in COLT UI retains messages from previous session (e.g., after the page was reloaded). You can turn this behavior off here.

Redirection Snippet

To launch phonegap app on the device, place this code in phonegap main HTML document, so that it will load processed code from COLT local web server.

Main Document URL

If you need the link to your app that you can send to someone, you could copy it from browser window, or, if you launch the app manually, you could get it here.

Sublime commands

You can find all the COLT commands in Tools menu:

Start COLT

Start COLT application. You need this command in case you want to tweak project settings, otherwise "Run with COLT" command will start COLT and initiate live session for you.

Run with COLT (Browser/Node.js/node-webkit)

This command starts COLT if it's not open yet, creates autogenerated.colt project file in currently open Sublime folder and sets the document in currently opened Sublime tab as "Main Document".

Reload in COLT

Reloads the page/app. The command only works if live session is active. Use this when you need to reload the page, or node.js script.

Disable/Enable Autosave

When autosave is on, the text will be saved automatically after every keystroke.

This mode can be useful when editing CSS and HTML.

Show Value in Console

Select variable or expression and invoke this command to see selection last known value in Sublime console.

This command makes you forget about console.log() when you only want to check variable value. The runtime environment does console.log() for you and you can see the value in browser or node-webkit Developers Tools where you can inspect the value in detail.

This command allows to check the values in functions that were already executed.

Execute Function

You can run particular function on demand with this command. For example, you can modify function logic and call it again to see the changes. Important - the function will receive the same arguments that it had the last time it was called by your application. The arguments will be undefined if the function was never called before.

Show JSDoc

Place the caret in the function or function name in the call, and invoke this command - function documentation will pop up.

Go to Declaration

This is useful when you need to see the code of the function that is called. Place the caret in the function name in the call and press F3. Sublime will open the document with relevant function.

Show Last Errors

If the application has syntax or runtime errors, this command will pop up the list of lines, where these errors happened. Use keyboard arrows to navigate this list and switch to corresponding documents.

Function call counts

You can see if, or how many times, each function was called so far. The count is displayed in the grey circle next to first line of function with the numbers 1-9 (or infinity for higher call counts).

Function call counts are reset when the session or the application is restarted.

Runtime and syntax error display

If your code has runtime or syntax errors, there will be red error icon in the gutter next to it. If you place the caret anywhere in this line, error details will be shown in Sublime Text status bar below.

Autocompletion

COLT also improves variables and object properties autocompletion.

Live session

Live session is the state of your application when it updates itself in response to saving project files without restarting.

Starting Live Session

When you start the session from your IDE ("Run with COLT") or click green "lighning bolt" icon in COLT, the following happens.

  1. The session begins, COLT-servers is given project file, and copies required files to temporary folder (out).
  2. Transformed copy of main document is created (live.js script is added, etc).
  3. COLT waits for request for main document.
  4. Once main document is requested, COLT http server starts serving scripts and assets, copying and transforming them before they are loaded into main document.
  5. Main document creates socket/ajax connection to COLT.

Once connection is made, the "lightning bolt" in COLT UI turns green and code editors with COLT plugins will also indicate that live session has started.

Session termination

The session might be closed due to various reasons:

  1. The newer session with same parameters was started (project file or main document)
  2. There was browser or other runtime error that caused the connection with COLT to close.
  3. COLT UI has triggered "Close Session" command (green "lightning bolt" was clicked second time)

HTML updates algorithm

Modern application development assumes using multiple html files. One Page Application. Every tag in your project gets globally unique identifier, that allows to locate it in your application no matter how did it get there. So when you change the tag in any file in the project, its new state will be reflected at run-time. To help you see the changes, this tag will be temporary highlighted with grey border. Keep in mind that these HTML and JS updates work across any connected browsers and devices, mobiles, TVs, etc.

Angular Support

Directives are recompiled after HTML live-update. In other words, if you changed your directive, such as {{myData}}, you will see your data from $scope instead of curly brackets after update. Our HTML recompilation algorithm is smart enough to handle things like ng-repeater. To better see the effect of your changes we highlight affected element with red border.

HTML updates and directives recompilation can work with dynamically added HTML. You can edit the template already loaded into main page through ng-router and see the end result immediately.

Live-Proxy

You can specify web URL as main document in the project settings. In this case COLT will load web pages and resources from web server, and transform them to make it work in live mode on the fly. As soon as the resource is modified on the server, COLT will react and deliver updates to all connected clients. Now it does not matter what technology or languages are you using on serverside. Sublime will still get updates, compile, and trace to the console. You only need to make sure your files are synced to server using your favorite existing solution. We expect this to help you use COLT and livecoding in large projects with complex serverside.

Run COLT from Console

You can launch the jar using shell, using the path to project xml as the argument.

Example:

$ java jar colt.jar myProject.colt

This mode is perfect for server setup for using COLT as live-proxy.
